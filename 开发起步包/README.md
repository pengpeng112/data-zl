## 关键硬信息速查（详见 `02`）

**连接**
```
跳板机:    ssh -p 40022 -i ~/.ssh/id_ed25519_ai root@10.10.8.53
源 ODS:    ods/ods123@10.10.8.216:1521/orcl          (只读，探库主目标)
前置机:    dboper/dboper123@10.10.8.74:1521/orcl     (标准化层，参考)
DBLINK:    sjzx  (前置机 → 源 ODS)
Oracle客户端: /opt/oracle/instantclient_21  (thick 模式必需)
```

**必须避的坑**
1. ⚠️ 本机 Windows 不能直连数据库，所有 DB 操作经 8.53 跳板机。
2. ⚠️ 源库 Oracle 旧版，`oracledb` 必须 thick 模式 + instantclient_21，否则 `DPY-3010`。
3. ⚠️ `HIS.LAB_RESULT` 约 1 亿行，查询必须用 `TEST_NO` 子查询限定，禁全表扫描。
4. ⚠️ `HIS.OPERATION.OPER_ID` 全为 NULL，手术关联用 `SM.MED_OPERATION_NAME`。
5. ⚠️ HIS.EXAM_MASTER.EXAM_CLASS 存中文（'CT'/'磁共振'），非字典内码。
6. ⚠️ 大表（>1000 万行）只采元数据；探库夜间执行并加超时。
