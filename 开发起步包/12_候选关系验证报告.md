## 2. 本轮数据库验证结论

| ID | 候选关系 | 验证结果 | 结论 |
|---|---|---|---|
| C01 | `MTL.EPR_INPATIENT_DOCINDEX → HIS.PAT_VISIT` (`P_CLINIC_ID + P_INPATIENT_TIMES`) | 239,863 行，2,876 孤儿（1.20%） | 可作为旧 EMR 文书直连住院关系，少量历史孤儿 |
| C02 | `MTL.EMRSD_CONTENT → MTL.EPR_INPATIENT_DOCINDEX` (`UHID`) | 100,000 样本，1 孤儿 | 强关系，段落内容可直接按 `UHID` 追溯文书索引 |
| C03 | `SM.MED_ANESTHESIA_PLAN → SM.MED_OPERATION_MASTER` (`PATIENT_ID+VISIT_ID+OPER_ID`) | 132,818 行，1,052 孤儿（0.79%） | 可作为麻醉计划挂手术主记录关系 |
| C04 | `SM.MED_OPERATION_MASTER.SURGEON → HIS.STAFF_DICT.EMP_NO` | 138,551 行，92,300 孤儿（66.6%） | 不可按 `EMP_NO` 强关联，`SURGEON` 可能存姓名/异构编码 |
| C05 | `SM.MED_OPERATION_MASTER.ANESTHESIA_DOCTOR → HIS.STAFF_DICT.EMP_NO` | 138,551 行，23,554 孤儿（17.0%） | 可参考但不稳定，需查 SM 人员字典或姓名匹配 |
| C06 | `MTL.EPR_INPATIENT_DOCINDEX.P_DEPT_CODE → HIS.DEPT_DICT.DEPT_CODE` | 239,863 行，0 孤儿 | 强关系，旧 EMR 文书科室可直接挂 HIS 科室字典 |
| C07 | `SM.MED_OPERATION_SCHEDULE → SM.MED_OPERATION_MASTER` (`PATIENT_ID+VISIT_ID+OPER_ID`) | 192,538 行，74,699 孤儿（38.8%） | 排班/计划表不能直接当手术事实；需按状态/已执行子集拆分 |
| C08 | `SM.MED_CUSTOM_DATA → SM.MED_OPERATION_MASTER` | 5,000 个手术键限定，命中 1,030,763 行，覆盖 4,178 个手术键 | 关系存在但为超多明细，必须按手术键限定 |
| C09 | `HIS.OPERATION → HIS.PAT_VISIT` (`PATIENT_ID+VISIT_ID`) | 528,729 行，16 孤儿 | HIS 侧手术可挂住院，但不能与 SM 手术一一对应 |
| C10 | `HIS.OPERATION` 同住院是否有 `SM.MED_OPERATION_MASTER` | 528,729 行，其中 247,724 行同住院存在 SM 手术 | 仅说明同一次住院存在 SM 手术，不构成一对一关系 |
| C11 | `HIS.DIAGNOSIS → HIS.PAT_MASTER_INDEX` (`PATIENT_ID`) | 3,330,143 行，81 孤儿 | 强关系；已可由 `DIAGNOSIS→PAT_VISIT→PMI` 间接获得 |
| C12 | `HIS.CLINIC_MASTER.VISIT_DEPT → HIS.DEPT_DICT.DEPT_CODE` | 5,704,241 行，0 孤儿 | 门诊就诊科室强关系 |
| C13 | `SM.MED_OPERATION_MASTER → HIS.PAT_MASTER_INDEX` (`PATIENT_ID`) | 138,551 行，267 孤儿（0.19%） | 手麻患者主索引关系成立 |

## 4. 不应直接提升的候选

| 关系 | 原因 | 后续处理 |
|---|---|---|
| `SM.MED_OPERATION_MASTER.SURGEON → HIS.STAFF_DICT.EMP_NO` | 孤儿率 66.6%，字段可能不是 EMP_NO | 查 SM 人员字典，或改用姓名/用户表映射 |
| `SM.MED_OPERATION_MASTER.ANESTHESIA_DOCTOR → HIS.STAFF_DICT.EMP_NO` | 孤儿率 17.0% | 同上，需找 SM 人员主数据 |
| `SM.MED_OPERATION_SCHEDULE → SM.MED_OPERATION_MASTER` | 孤儿率 38.8%，排班表含未执行/取消/历史计划 | 加状态过滤后重验 |
| `HIS.OPERATION ↔ SM.MED_OPERATION_MASTER` | 只有 46.9% HIS.OPERATION 同住院存在 SM 手术，且无可用 `OPER_ID` | 只作为同住院弱关联，不做手术一对一 |
| `SM.MED_CUSTOM_DATA → SM.MED_OPERATION_MASTER` | 命中但明细量极大 | 可作为明细扩展表，查询必须按手术键限定 |
