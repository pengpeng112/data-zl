## 0. 执行边界与原则

### 0.1 当前目标

第一阶段只交付 **只读数据资产门户 MVP**：让信息科能浏览表、字段、关系和关系路径。SQL 沙箱、AI 自动执行、探库 Worker、Dify 自动化均放到后续阶段。

### 0.2 安全边界

- 源库 `8.216` / HIS 源端只读，只允许 `SELECT`，禁止任何 DML/DDL。
- P1 不直连源库，只读 `data_asset` 元数据库。
- 外网 AI 只接收脱敏元数据，不接收患者姓名、身份证、电话、地址和业务明细数据。
- SQL 沙箱未完成前，不在页面开放任意 SQL 执行能力。
- 密码、Key、连接串不得写入代码或文档；统一使用 `.env` / 运维密钥渠道。

### 0.3 执行角色

| 角色 | 职责 |
|---|---|
| AI/开发者 | 按本计划修改代码、补测试、更新文档、记录状态 |
| 用户/项目负责人 | 确认范围、优先级、登录策略、上线节奏、外部依赖信息 |
| 信息科/业务验收人 | 验收 M1 页面、反馈字段/关系展示是否够用 |
| 运维/DBA | 确认 8.83 部署、备份、端口、Nginx、账号权限 |

### 0.4 计划计时方式

由于具体人员与排期未固定，本文采用相对时间：`D0` 为用户确认本计划并授权开工当天。

## 3. 功能需求清单

### 3.1 P1 必须实现：只读资产门户 MVP

| 功能 | 用户价值 | 核心能力 | 验收口径 |
|---|---|---|---|
| 资产总览 | 快速了解当前资产规模 | 展示表数、字段数、关系数、业务域数；展示业务域分布图 | 首页显示 865/26894/47/19 或与当前 `data_asset` 一致 |
| 表资产清单 | 信息科可自助找表 | 按 keyword、schema、domain 查询表；分页、排序；展示表名、注释、字段数、来源域 | 搜索 `PAT_VISIT` 能命中；分页不一次加载全部字段 |
| 表详情 | 快速理解一张表的字段和上下游 | 展示表基本信息、字段列表、字段类型、nullable、字段注释、上下游关系、验证等级 | 打开 `HIS.PAT_VISIT` 可看到字段和 `PAT_MASTER_INDEX` 等关联 |
| 字段搜索 | 按业务词查字段 | 支持字段名/注释 keyword 查询，后端分页返回 | 搜索住院号、检查、检验等关键词能返回相关字段 |
| 关系路径查询 | 看清两张表怎么关联 | 输入起点表/终点表，返回最短路径；图形展示节点、边、join 条件、验证状态 | `HIS.PAT_VISIT -> HIS.PAT_MASTER_INDEX` 返回 1 跳路径并可视化 |
| 关系明细查看 | 判断关系是否可信 | 展示 join 条件、验证等级、状态、孤儿率/样本指标、备注 | 正式关系可显示 `validation_level/status/metrics/note` |
| AI 上下文导出 | 给外网 AI 提供安全上下文 | 选择表后导出脱敏 JSON/文本，包含表、字段、关系、注意事项，不含患者明细 | 导出内容只有元数据；能复制/下载；附 AI 使用提示 |
| 基础权限策略 | 控制门户访问入口 | 保留 pure-admin 登录或按用户确认做内网免登录 | 策略与决策项 D-02 一致；未确认时默认保留登录页 |
| 健康检查与错误提示 | 降低联调和上线排错成本 | `/health` 检查服务和数据库；前端统一错误提示 | 数据库异常时能明确提示，不出现空白页 |

### 3.2 P1 暂不实现：明确排除项

| 暂不实现功能 | 原因 | 计划阶段 |
|---|---|---|
| 任意 SQL 在线执行 | 风险较高，需要 SQL 沙箱、只读账号、审计和超时保护 | P2 |
| AI 直接调用数据库 | 安全风险高，外网 AI 只能看脱敏元数据 | 不开放；P2 仅支持候选 SQL |
| 自动探库和自动刷新元数据 | 需要连 8.216/HIS 源端，必须确认账号、窗口和限流 | P3 |
| Dify 自动影响分析 | 依赖 Dify API/Key/Workflow | P3 |
| CDA 映射展开 | 当前阶段已明确暂不展开 | P3 之后另定 |
| 患者明细查询 | 本平台当前定位是元数据资产门户，不做业务明细查询 | 暂不做 |
| 数据修改/资产人工编辑 | P1 只读，避免误改资产底座 | P2/P3 后按审批加入 |

### 3.4 P2 后续功能：AI 协作与 SQL 沙箱

| 功能 | 用户价值 | 安全要求 |
|---|---|---|
| Prompt 模板 | 让外网 AI 基于表关系生成 SQL/视图草稿 | 输入只包含脱敏元数据 |
| SQL 工作台 | 内网人工执行只读分析 SQL | 只允许 `SELECT`，强制超时、行数限制和审计 |
| SQL 风险拦截 | 防止误跑危险 SQL 或大表扫描 | sqlglot 解析 + 只读账号 + 只读事务 + 黑白名单 |
| 视图草稿管理 | 保存 AI 生成的视图/SQL 草稿 | 草稿需人工确认，不自动写入源库 |
| SQL 审计 | 可追踪谁执行了什么 | 记录用户、SQL、耗时、行数、状态、命中规则 |
| AI 反馈闭环 | 评估 AI 生成质量 | 记录采纳/拒绝/修改原因 |

### 3.5 P3 后续功能：探库自动化与数据治理

| 功能 | 用户价值 | 约束 |
|---|---|---|
| 元数据定时采集 | 自动发现表/字段/视图变化 | 夜间执行，只采元数据，大表不扫数据 |
| 快照差异检测 | 识别新增/删除/变更字段和视图 DDL | 变更入库，人工确认后影响正式资产 |
| 资产包同步 | 保持 `data_asset` 与 `数据资产_资产包/` 一致 | 同步前预览差异，确认后写入 |
| Dify 影响分析 | 自动解释元数据变更影响 | 依赖 Dify API/Key/Workflow |
| 字段语义检索 | 按业务概念找相似字段 | 基于 pgvector，向量内容仅元数据 |
| owner/术语/质量规则 | 补充轻量数据治理能力 | P1 不阻塞，P3 后逐步建设 |

### 3.6 非功能需求

| 类别 | 要求 |
|---|---|
| 安全 | 不存明文密码；不导出患者明细；源库只读；危险 SQL 拦截 |
| 性能 | 表/字段查询走后端分页；默认 `page_size<=50`；大结果截断 |
| 可用性 | 页面不能因单接口失败整体空白；错误提示可理解 |
| 可维护性 | 后端 API 有 Swagger；关键逻辑有测试；新增表结构走迁移记录 |
| 可审计 | P2 起 SQL 执行、AI 草稿和资产变更均需记录审计 |
| 可部署 | 支持离线打包，经本机 -> 8.53 -> 8.83 部署 |
| 可回滚 | 每次上线保留上一版本前端、后端、Nginx 配置 |

### 6.3 SQL 沙箱硬规则

- 只允许单条 `SELECT` / `WITH ... SELECT`。
- 默认 `LIMIT 100` 或等价行数截断；下载能力另行审批。
- 设置数据库级超时；接口层也设置超时。

- 外网 AI 不直接调用 `run_sql`，只能生成候选 SQL；内网人工确认后执行。
