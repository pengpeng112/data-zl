## 二、探库前待确认的开放问题

这些问题直接影响关系图谱质量，建议探库启动前先确认或边探边补：

1. **各业务系统数据库连接信息**：HIS/EMR/LIS/PACS/手麻/输血/院感/无纸化分别在哪台库、什么账号？是否都汇聚到 8.216，还是分散在多台库？
2. **患者主索引（EMPI）方案**：✅ 已发现 `PORTAL_EMPI` 库（`PATIENT_INFO` + `PATIENT_INFO_EQUAL/IRREGULARITY/SIMILARITY` + `PATIENT_MATCH_RULE`）。待确认：跨系统患者 ID 如何经这些匹配规则表统一；`PATIENT_ID` 是否跨系统一致。
3. **科室 / 人员主数据映射**：`DEPT_DICT`、`STAFF_DICT` 是否为全院统一主数据？各系统科室/人员编码是否一致？
4. **`HIS.DOC_REGISTER_EMP_MATCH` 缺失**：医师执业注册匹配表在数据中心 8.216 未发现（仅 HIS 生产库）。接入生产库后补校验执业证书号来源。
5. **Oracle 前置机 8.74 详细版本**：11g/12c/19c？影响 SQL 兼容性（`FETCH FIRST` 等，旧版用 `ROWNUM <= N`）。
6. **增量时间字段**：各源系统统一的更新时间戳字段是什么？（关系图谱阶段非必须，视图/抽取阶段需要）
7. **`V_EMR_*` 视图之外**，`ODS` 下是否还有其他未登记视图或原始表？
