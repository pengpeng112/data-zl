## 1. 总体结论

`29_系统功能对照复核与整改计划.md` 可作为当前整改入口，但质量模块需要追加一条明确主线：**把现有“资产元数据质量检查”升级为“医院级数据治理质控规则库”**。

当前代码已有基础：

| 能力 | 当前代码 | 结论 |
|---|---|---|
| 质量规则表 | `backend/app/models/quality.py::QualityRule` | 已有，但字段不足 |

核心缺口：现有 `QUALITY_RULES_SEED` 只检查关系孤儿率、中文名缺失、字段注释缺失、元数据过期等资产治理问题，尚未支持用户提出的五类医院质控规则：`UNIQUE / COMPLETE / STANDARD / RELATION / ACCURACY`，也没有 `CHECK_SCOPE`、`CONSTRAINT_LEVEL`、`CHECK_SQL`、结果样例、规则模板、问题整改闭环。

## 2. 规则体系并入方式

### 2.1 五类一级规则

系统规则分类统一使用：

| 编码 | 名称 | 代码落地字段 | 用途 |
|---|---|---|---|
| `UNIQUE` | 唯一性规则 | `rule_category` | 主键、业务主键、组合键重复 |
| `COMPLETE` | 完整性规则 | `rule_category` | 必填、条件必填、主从完整 |
| `STANDARD` | 规范性规则 | `rule_category` | 长度、值域、格式、精度、编码 |
| `RELATION` | 关联性规则 | `rule_category` | 主从关系、跨表、跨系统追溯 |
| `ACCURACY` | 准确性规则 | `rule_category` | 时间、金额、诊疗、状态逻辑 |

保留现有 `rule_type` 作为二级规则类型，例如 `primary_key_duplicate`、`required_empty`、`value_domain_invalid`、`orphan_record`、`time_logic_error`。

### 2.2 检查层级

新增字段 `check_scope`：

| 编码 | 名称 | 示例 |
|---|---|---|
| `TABLE_INNER` | 表内质控 | ID 重复、必填为空、日期逻辑错误 |
| `TABLE_RELATION` | 表间质控 | 子表关联不到主表 |
| `SYSTEM_CROSS` | 跨系统质控 | HIS 手术与 EMR 手术记录不一致 |
| `BUSINESS_LOGIC` | 业务逻辑质控 | 出院时间早于入院时间 |

### 2.3 约束级别

新增字段 `constraint_level`：

| 编码 | 名称 | 处理方式 |
|---|---|---|
| `HARD` | 硬性约束 | 不允许入库或不允许上报 |
| `SOFT` | 柔性约束 | 允许入库，但产生预警 |
| `WARN` | 提醒规则 | 仅提示，不影响主流程 |
| `INFO` | 信息规则 | 仅统计趋势 |

## 4. 规则执行器设计

### 4.1 执行模式

保留并扩展 `execution_mode`：

| 模式 | 说明 | 当前状态 |
|---|---|---|
| `metadata_only` | 只基于资产元数据检查 | 已实现 |
| `sql_template` | 执行配置 SQL，统计异常行 | 待实现 |
| `source_probe` | 连接源库采样/探查后检查 | 待实现，依赖源库连接整改 |
| `aggregate_compare` | 主表金额与明细汇总等聚合比对 | 待实现 |

### 4.2 SQL 安全约束

质控 SQL 只能是只读查询：

1. 只允许 `SELECT` / `WITH SELECT`。
2. 禁止 `INSERT/UPDATE/DELETE/MERGE/DDL/CALL/EXEC`。
3. 禁止多语句。
4. Oracle 11g 限制行数使用 `ROWNUM <= N`，不能用 `FETCH FIRST`。
5. `HIS.LAB_RESULT`、`INP_BILL_DETAIL`、`ORDERS` 等大表必须带主键、时间范围或子查询限制，禁止全表扫描。
6. `sample_data` 入库前必须脱敏姓名、身份证、电话、地址、明文密码。

### 4.3 建议新增服务

| 文件 | 职责 |
|---|---|
| `backend/app/services/quality_rule_engine.py` | 规则分发、SQL 安全扫描、结果汇总 |
| `backend/app/services/quality_sql_runner.py` | 执行只读 SQL、限制行数、脱敏样例 |
| `backend/app/services/quality_templates.py` | 唯一性、完整性、规范性、关联性、准确性 SQL 模板生成 |
| `backend/app/services/data_masking.py` | 统一脱敏，复用运维/AI/质量模块 |

| `R_INP_002` | 住院唯一标识不能为空 | 住院 | `COMPLETE` | `TABLE_INNER` | `HARD` |
| `R_DIAG_001` | 主要诊断只能有一条 | 诊断 | `UNIQUE` | `TABLE_INNER` | `HARD` |
| `R_ORDER_001` | 医嘱必须关联住院患者 | 医嘱 | `RELATION` | `TABLE_RELATION` | `HARD` |
