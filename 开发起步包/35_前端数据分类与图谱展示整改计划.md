# 前端数据分类与图谱展示整改计划

## 1. 用户补充描述

用户反馈：当前程序中的数据分类很不清晰，希望前端界面的数据表按照“系统、表空间、表、字段”进行分析和展示；各个系统之间可以建立关联关系；展示时尽量使用中文并带有库名；同时希望图谱关系显示得更高端大气。要求先生成整改 Markdown 计划，并将上述描述补充进去。

## 2. 当前问题

1. 数据资产列表缺少清晰层级，用户难以从系统视角理解表属于哪个业务系统、库、Schema/表空间。
2. HIS、数据中心、EMR、LIS、PACS、护理、手麻、费用、病案等系统之间的关联没有以跨系统方式表达。
3. 表名、字段名、关系说明偏技术化，中文名、库名、来源说明不够突出。
4. 图谱展示偏基础，缺少分层、筛选、聚合、关系等级和业务域表达。
5. A/B/C/D 关系、核心/排除/待确认表、活跃度、主键验证等二次优化结果尚未充分进入前端交互。

## 3. 整改目标

将数据资产前端改造成“系统 -> 库/表空间 -> 表 -> 字段”的四级资产浏览结构，并支持跨系统关系图谱。

核心目标：

- 让用户先看系统，再看库名/Schema，再看表，再看字段。
- 所有资产展示优先使用中文名称，同时保留英文库名、Schema、表名、字段名。
- 表关系支持同系统内关系和跨系统关系。
- 图谱只默认展示 A 级确定关系，B/C/D 关系进入复核视图。
- 图谱展示更专业：支持业务域分组、系统分组、关系等级、节点展开、搜索定位、路径分析。

## 4. 前端信息架构

### 4.1 系统层

展示医院业务系统，例如：

- 数据中心
- HIS
- EMR/电子病历
- LIS/检验
- PACS/检查影像
- 护理
- 手麻
- 费用/医保
- 病案
- 字典/主数据

每个系统卡片展示：

- 系统中文名
- 系统编码
- 库名/数据源数量
- 表数量
- 字段数量
- A 级关系数量
- 待确认表数量
- 最近元数据更新时间

### 4.2 库/表空间层

HIS 示例：

- 库名：`his_ready_10_10_10_15`
- 连接：`10.10.10.15:1521/his`
- Schema/表空间：`MEDREC`、`ORDADM`、`LAB`、`EXAM`、`PHARMACY`、`INPBILL`、`OUTPBILL`、`COMM`

界面展示字段：

- 库名
- Schema/表空间
- 中文业务域
- 表数量
- 核心表数量
- 排除表数量
- 待确认表数量
- 数据量合计
- 最近活跃时间

### 4.3 表层

表列表应支持按以下维度筛选：

- 系统
- 库名
- Schema/表空间
- 业务域
- 表类型
- 活跃度
- 是否核心表
- 是否纳入治理
- 确认状态
- 来源文件名称

表卡片或表格建议展示：

- 表中文名
- 库名.Schema.表名
- 业务域
- 表类型
- 数据粒度
- 总行数
- 最近数据时间
- 活跃度
- 候选主键
- 关联字段
- 来源文件名称
- 确认状态

### 4.4 字段层

字段详情应展示：

- 字段中文名/字段含义
- 英文字段名
- 数据类型、长度、精度
- 是否主键候选
- 是否关联字段
- 是否敏感字段
- 是否质量分析字段
- 建议质量规则
- 字典映射情况
- 文档备注
- 实际库备注
- 文档与实际库是否一致

## 5. 图谱展示整改方案

### 5.1 图谱分层

建议提供三种图谱视图：

1. 系统关系图：展示 HIS、EMR、LIS、PACS、护理、手麻、数据中心之间的关系。
2. 业务域关系图：展示患者、就诊、诊断、医嘱、检查、检验、费用、药品、文书之间的关系。
3. 表级 ER 图：只展示 A 级确定关系，支持展开到字段级。

### 5.2 图谱视觉设计

图谱应避免普通散点式节点堆叠，建议采用：

- 系统分区背景：不同系统用不同分组区域。
- 业务域颜色：患者、就诊、医嘱、检验、检查、费用等使用稳定色。
- 节点大小表达重要性：核心表更大，字典表更小。
- 边样式表达关系等级：A 实线，B 虚线，C 红色警示，D 灰色待确认。
- 默认隐藏 B/C/D，只在复核模式显示。
- 鼠标悬停显示关系证据：覆盖率、孤儿数、验证来源、风险说明。
- 支持搜索表名/中文名后自动聚焦节点。
- 支持一键查看“患者全链路”“住院全链路”“门诊全链路”“检验检查链路”“费用链路”。

### 5.3 推荐技术实现

优先方案：

- 表格与筛选：Element Plus `ElTable`、`ElTree`、`ElTabs`、`ElDrawer`
- 图谱：AntV G6 或 Cytoscape.js
- 层级资产树：系统 -> 库名 -> Schema -> 表
- 详情面板：右侧 Drawer 显示表、字段、关系、质量规则

如需更强视觉效果，图谱可采用：

- AntV G6：适合企业级关系图、分组、边样式、交互。
- Cytoscape.js：适合复杂网络、布局算法和关系分析。

不建议初期使用纯 Mermaid 作为主交互图谱；Mermaid 可用于报告和静态预览。

## 7. 页面整改清单

### P0：资产浏览重构

- 新增“系统资产总览”页面。
- 新增“系统 -> 库名 -> Schema -> 表”左侧资产树。
- 表列表增加库名、Schema、中文名、业务域、活跃度、确认状态。
- 表详情页增加字段、关系、活跃度、主键验证、质量规则 Tab。

### P1：关系复核与图谱

- 图谱默认只展示 A 级关系。
- 新增 B/C/D 关系复核列表。
- 图谱节点支持按系统、业务域、表类型筛选。
- 点击节点显示表详情，点击边显示验证证据。

### P2：中文化与来源增强

- 所有页面优先展示中文名。
- 英文库名、Schema、表名作为副标题。
- 增加来源文件名称、来源章节、文档与实际库是否一致。
- 缺失备注统一显示“需要人工确认”，禁止编造。

### P3：质量分析入口

- 在字段详情中展示建议质量规则。
- 支持按 P1/P2/P3 质量优先级筛选字段。
- 为主键、外键、时间、状态、编码字段提供质量分析入口。

## 8. 验收标准

- 用户能从系统层快速定位到具体库、Schema、表和字段。
- 每张核心表能看到中文名、库名、Schema、业务域、表类型、数据粒度和来源文件。
- A 级关系能在图谱中清晰展示，B/C/D 不混入正式 ER 图。
- 图谱支持搜索、筛选、展开、查看证据。
- 排除表和待确认表不与核心表混杂展示。
- 缺失备注、缺失来源章节、未验证关系都能清晰标记为“需要人工确认”。
