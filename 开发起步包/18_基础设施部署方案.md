## 3. 元数据库初始化（P1，纯 SQL，不依赖离线包）

### 3.1 建库与账号分离

经跳板机 → 8.83，以 `postgres` 超级用户执行：

```sql
-- 库
CREATE DATABASE data_asset
  WITH OWNER = postgres ENCODING 'UTF8';

\c data_asset

-- schema
CREATE SCHEMA asset AUTHORIZATION postgres;

-- 扩展占位（pgvector 装好后执行；此处先注释，P3 完成后回填）
-- CREATE EXTENSION vector WITH SCHEMA asset;

-- 角色：账号分离（01 §16.2.1）
CREATE ROLE asset_app    WITH LOGIN PASSWORD '<APP_PASS>'    ;  -- 应用读写
CREATE ROLE asset_ro     WITH LOGIN PASSWORD '<RO_PASS>'     ;  -- 只读/AI工具API
CREATE ROLE asset_backup WITH LOGIN PASSWORD '<BACKUP_PASS>' ;  -- 备份

-- 权限：asset_app 读写 asset schema
GRANT USAGE ON SCHEMA asset TO asset_app, asset_ro, asset_backup;
GRANT ALL ON SCHEMA asset TO asset_app;
ALTER DEFAULT PRIVILEGES IN SCHEMA asset
  GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO asset_app;
ALTER DEFAULT PRIVILEGES IN SCHEMA asset
  GRANT SELECT ON TABLES TO asset_ro, asset_backup;

-- 备份账号：全库只读（用于 pg_dump）
GRANT pg_read_all_data TO asset_backup;
```

> `<APP_PASS>` / `<RO_PASS>` / `<BACKUP_PASS>` 三套密码：由信息科设定，不写入文档（脱敏占位）。

### 3.2 资产元数据表 DDL（基于资产包 CSV 实际列）

```sql
-- 表清单（对应 tables.csv，11 列）
CREATE TABLE asset.asset_tables (
    id               BIGSERIAL PRIMARY KEY,
    schema_name      TEXT NOT NULL,
    table_name       TEXT NOT NULL,
    comment          TEXT,
    row_count_stats  TEXT,
    column_count     INT,
    domain           TEXT,
    grain            TEXT,
    pk               TEXT,
    confidence       TEXT,
    note             TEXT,
    source           TEXT,
    UNIQUE (schema_name, table_name)
);
CREATE INDEX idx_asset_tables_domain ON asset.asset_tables(domain);

-- 字段清单（对应 columns.csv，8 列）
CREATE TABLE asset.asset_columns (
    id           BIGSERIAL PRIMARY KEY,
    schema_name  TEXT NOT NULL,
    table_name   TEXT NOT NULL,
    column_id    INT,
    column_name  TEXT NOT NULL,
    data_type    TEXT,
    length       INT,
    nullable     TEXT,
    comment      TEXT
);
CREATE INDEX idx_asset_columns_tbl ON asset.asset_columns(schema_name, table_name);

-- 关系（对应 relationships.csv，14 列）
CREATE TABLE asset.asset_relations (
    id                 BIGSERIAL PRIMARY KEY,
    rel_id             INT,
    domain             TEXT,
    from_table         TEXT,
    from_columns       TEXT,
    to_table           TEXT,
    to_columns         TEXT,
    join_condition     TEXT,
    cardinality        TEXT,
    confidence         TEXT,
    validation_level   TEXT,
    validation_status  TEXT,
    validation_metrics TEXT,
    note               TEXT,
    validation_note    TEXT,
    UNIQUE (rel_id)
);

-- 导入记录（追溯每次导入）
CREATE TABLE asset.import_log (
    id           SERIAL PRIMARY KEY,
    object_type  TEXT,   -- tables/columns/relations
    row_count    INT,
    imported_at  TIMESTAMPTZ DEFAULT now(),
    source_file  TEXT
);
```

> 列名规避 PG 关键字：CSV 的 `schema`→`schema_name`、`table`→`table_name`、`column`→`column_name`。

## 4. 资产包导入（P2，依赖 P1，不需离线包）

### 4.1 BOM 处理

资产包 CSV 为 **UTF-8-BOM**（便于 Excel），`\copy ... HEADER` 会把 BOM 算进首列名。导入前在 8.83 上去 BOM：

```bash
cd /home/deploy/data
for f in tables.csv columns.csv relationships.csv; do
  sed -i '1s/^\xEF\xBB\xBF//' "$f"
done
```

### 4.2 导入命令（psql \copy，客户端读本地文件）

```sql
\c data_asset
TRUNCATE asset.asset_tables, asset.asset_columns, asset.asset_relations RESTART IDENTITY;

\copy asset.asset_tables(schema_name,table_name,comment,row_count_stats,column_count,domain,grain,pk,confidence,note,source) FROM '/home/deploy/data/tables.csv' WITH (FORMAT csv, HEADER true)
\copy asset.asset_columns(schema_name,table_name,column_id,column_name,data_type,length,nullable,comment) FROM '/home/deploy/data/columns.csv' WITH (FORMAT csv, HEADER true)
\copy asset.asset_relations(rel_id,domain,from_table,from_columns,to_table,to_columns,join_condition,cardinality,confidence,validation_level,validation_status,validation_metrics,note,validation_note) FROM '/home/deploy/data/relationships.csv' WITH (FORMAT csv, HEADER true)

INSERT INTO asset.import_log(object_type,row_count,source_file) VALUES
 ('tables',(SELECT count(*) FROM asset.asset_tables),'tables.csv'),
 ('columns',(SELECT count(*) FROM asset.asset_columns),'columns.csv'),
 ('relations',(SELECT count(*) FROM asset.asset_relations),'relationships.csv');
```

**预期计数校验**：tables=865、columns≈26894、relations=47。

### 6.1 离线包

从华为开源镜像站下 **openEuler 22.03 LTS SP2** 的 nginx 及依赖 rpm（含 `gd`/`libwebp` 等），清单待 `dnf download --resolve` 在本机（若有 openEuler 环境）或直接 URL 抓取：

```text
nginx-*.oe2203sp2.x86_64.rpm
（依赖：gd, libXpm, libwebp, ... 一并下载）
```

> 若本机非 openEuler 无法用 dnf 解析依赖，则手动从华为源目录抓全依赖链。

### 6.2 安装（8.83）

```bash
cd /home/deploy/pkg/nginx
dnf localinstall -y ./*.rpm
nginx -v
systemctl enable --now nginx
```

## 12. 风险与回滚

| 风险 | 对策 |
|---|---|
| 建库影响现有业务库 | 仅 CREATE DATABASE，不触碰 ai_hms_db/dialysis；可 DROP DATABASE data_asset 回滚 |
| CSV 导入 BOM/编码错乱 | 预先 sed 去 BOM；\copy 失败不影响表结构 |
| pgvector 编译失败 | 已确认 gcc/make/头文件齐；失败可 make clean 重试或换预编译 rpm |
| Nginx 依赖缺失 | localinstall 前用 `dnf --disablerepo=* localinstall` 看缺哪些补齐 |
| 离线包下载不全 | 本机下载后校验大小/哈希再中转 |
