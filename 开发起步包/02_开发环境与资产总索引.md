## 2. 网络与连接

### 2.1 拓扑

```text
本机开发机（Windows）
    │ SSH
    ▼
10.10.8.53:40022  跳板机 / 欧拉工具机
    │
    ├── 10.10.8.216:1521/orcl  数据中心 ODS 汇聚库（只读，探库主目标）
    └── 10.10.8.74:1521/orcl   前置机 DBOPER（标准化层，仅参考）
```

### 2.2 连接硬约束

- 本机 Windows 不能直连 Oracle，所有 DB 操作经 8.53 跳板机。
- 8.216 Oracle 版本为 **11.2.0.4.0**，`oracledb` 必须 thick 模式 + `/opt/oracle/instantclient_21`，否则 `DPY-3010`。
- 旧版 Oracle 不支持 `FETCH FIRST`，抽样使用 `ROWNUM <= N`。
- 源库只读，仅允许 `SELECT`。任何 DML/DDL 必须人工确认，默认禁止。
- 连接账号和密码不要写入新代码仓库；本参考包内已有历史凭据仅供内部查阅。

### 2.3 跳板机命令

```bash
ssh -p 40022 -i ~/.ssh/id_ed25519_ai root@10.10.8.53
```

### 2.4 Python thick 模式模板（在 8.53 上执行）

```python
import os, oracledb

oracledb.init_oracle_client(
    lib_dir=os.environ.get("ORACLE_CLIENT", "/opt/oracle/instantclient_21")
)

conn = oracledb.connect(
    user=os.environ["ODS_USER"],
    password=os.environ["ODS_PASSWORD"],
    dsn="10.10.8.216:1521/orcl",
)
```

## 4. 核心主线与表结构入口

详细表结构与关联见 `09_数据资产_表结构与关联关系.md`，完整字段见 `数据资产_资产包/columns.csv`。

### 4.1 住院主线

```text
HIS.PAT_MASTER_INDEX(PATIENT_ID)
  └── HIS.PAT_VISIT(PATIENT_ID, VISIT_ID)
        ├── HIS.DIAGNOSIS(PATIENT_ID, VISIT_ID)
        ├── HIS.ORDERS(PATIENT_ID, VISIT_ID)
        ├── HIS.INP_SETTLE_MASTER / HIS.INP_BILL_DETAIL
        ├── SM.MED_OPERATION_MASTER / SM.MED_OPERATION_NAME
        ├── HIS.EXAM_MASTER / HIS.EXAM_REPORT(EXAM_NO)
        ├── HIS.LAB_TEST_MASTER / HIS.LAB_RESULT(TEST_NO)
        └── MTL.EPR_INPATIENT_DOCINDEX / EPR_INPATIENT / EMRSD_CONTENT
```

### 4.2 万能关联键

| 场景 | 键 |
|---|---|
| 患者 | `PATIENT_ID` |
| 住院 | `PATIENT_ID + VISIT_ID` |
| 门诊 | `OUTPATIENT_NUM` 或门诊业务表自身就诊键 |
| 检验 | `TEST_NO`，明细再加 `ITEM_NO/PRINT_ORDER` |
| 检查 | `EXAM_NO` |
| 手麻 | `PATIENT_ID + VISIT_ID + OPER_ID`（SM 内部） |
| 旧 EMR 文书 | `INP_NO = P_INPATIENT_NO`，`VISIT_ID = P_INPATIENT_TIMES`，文书内部 `UHID` |
| 新 EMR 文书 | `FILE_UNIQUE_ID`；与住院精确关联键待确认 |

## 5. 高风险数据坑

| 问题 | 正确处理 |
|---|---|
| `HIS.LAB_RESULT` 约 9631 万行 | 必须用 `TEST_NO` 子查询限定，禁止全表扫描 |
| `HIS.INP_BILL_DETAIL` 约 2.15 亿行 | 仅采元数据；业务查询必须按患者/时间/住院限定 |
| `HIS.ORDERS` 约 4103 万行 | 查询必须限定住院或时间范围 |
| `HIS.OPERATION.OPER_ID` 全为 NULL | 不用它关联手麻；手术主从用 `SM.MED_OPERATION_MASTER/NAME` |
| `HIS.EXAM_MASTER.EXAM_CLASS` 存中文 | 按 `'CT'`、`'磁共振'` 等中文过滤，不按字典内码 |
| `HIS.EXAM_REPORT` 无 `PATIENT_ID` | 必须经 `EXAM_NO` 关联 `EXAM_MASTER` |
| `HIS.DOC_REGISTER_EMP_MATCH` 在 8.216 缺失 | 仅 HIS 生产库，接入生产库后再校验 |
| `JHEMR` 与 `PAT_VISIT` 精确关联未确认 | 关系图谱中标为开放问题，不强行推断 |

## 6. 值域与固定字段

### 6.1 机构字段

历史文档里的 `YLV21051` 已过期，统一使用：

```sql
'49557032X' AS ORG_CODE,
'山东省第二人民医院' AS ORG_NAME,
'370104' AS DISTRICT_CODE
```

住院类 ID 常用：

```sql
'49557032X' || '|' || patient_id || '|' || visit_id AS ID
```

### 6.2 CDA 字典映射

```sql
nvl((SELECT zd.国标编码
       FROM cda.cda_dictionary zd
      WHERE zd.字典名称 = '性别代码'
        AND zd.院内编码 = pmi.sex), '9')
```

ICD-10 诊断标准化：

```text
CDA_DICTIONARY.字典名称 = 'ICD-10诊断编码'
CDA_DICTIONARY.系统标识 = 'HIS'
CDA_DICTIONARY.院标编码 = HIS.DIAGNOSIS.DIAGNOSIS_CODE
```

字典关联前应按编码聚合，避免一对多放大记录数。
